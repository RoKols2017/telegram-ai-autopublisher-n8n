# Portfolio Case

## What this MVP signals

This repository positions the author as an engineer who can:

- build application-layer AI automation workflows
- integrate OpenAI API and Telegram Bot API through `n8n`
- design prompts for multi-step generation
- separate infrastructure concerns from workflow concerns
- package technical work as a GitHub portfolio case

## Why this works as a portfolio MVP

- the workflow is documented as a production-style integration
- security and sanitization are treated as first-class engineering requirements
- the repository includes test helpers, troubleshooting, and deployment notes
- the structure is optimized for hiring visibility and reusable engineering artifacts
